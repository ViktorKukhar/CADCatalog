# Navigation System Implementation Guide

## Overview
This document describes the comprehensive navigation system built using Rails framework capabilities. The system provides isolated, clear, and uniform navigation implementation across the entire application.

## Architecture

### NavigationBuilder Service (`app/services/navigation_builder.rs`)
Core navigation logic using Rails framework APIs:

**Key Responsibilities:**
- Route generation using Rails routing helpers
- Navigation item structure standardization
- Active state determination from request context
- Breadcrumb trail generation
- Pagination link generation
- Hierarchical navigation support
- Authorization-based navigation filtering

**Core Methods:**
- `main_navigation()` - Primary application menu
- `user_navigation()` - Account dropdown menu
- `breadcrumbs()` - Breadcrumb trail for current page
- `resource_navigation(resource_name, id)` - Resource-specific links
- `category_navigation(categories)` - Filter/category sidebar
- `pagination_navigation(page, total, resource)` - Pagination links
- `mobile_navigation()` - Mobile hamburger menu
- `authorized_navigation(user)` - Permission-filtered menu

### NavigationHelper Module (`app/helpers/navigation_helper.rb`)
Rails view helper for rendering navigation:

**Key Methods:**
- `render_main_navigation()` - Render main menu
- `render_breadcrumbs()` - Render breadcrumb trail
- `render_pagination()` - Render pagination controls
- `render_tab_navigation()` - Render tab interface
- `render_category_navigation()` - Render sidebar categories
- `render_mobile_menu()` - Render mobile menu
- `render_resource_actions()` - Render action buttons

### ApplicationHelper Integration
The NavigationHelper is included in ApplicationHelper, making all navigation methods available in all views without explicit importing.

## Navigation Item Structure

### NavigationItem Struct
```ruby
NavigationItem = Struct.new(:label, :path, :icon, :active, :children, :badge)
```

**Properties:**
- `label` - Navigation item text
- `path` - URL generated by Rails routing helpers
- `icon` - Icon identifier for UI rendering
- `active` - Boolean indicating current/active state
- `children` - Array of nested NavigationItem objects
- `badge` - Optional badge value (e.g., count)

**Example:**
```ruby
NavigationItem.new(
  'Records',
  Rails.application.routes.url_helpers.records_path,
  'file-text',
  true,
  [/* children */],
  nil
)
```

## Framework Integration

### Rails Routing Helpers
Navigation uses only Rails' built-in routing helpers:
```ruby
Rails.application.routes.url_helpers.records_path
Rails.application.routes.url_helpers.edit_record_path(record)
Rails.application.routes.url_helpers.root_path
```

### Request Context
Navigation state determined from Rails request object:
```ruby
@current_request.path_parameters  # Extract controller/action
@current_request.path              # Current page path
@current_request.query_parameters  # URL query params
```

### View Rendering
Navigation helpers use Rails view helpers:
```ruby
link_to()           # Generate links
content_tag()       # Generate HTML tags
polymorphic_path()  # Resource-agnostic routing
```

## Navigation Patterns

### Single-Level Navigation
```ruby
main_nav = [
  NavigationItem.new('Home', root_path, 'home', true, [], nil),
  NavigationItem.new('Records', records_path, 'file', false, [], nil)
]
```

### Multi-Level Navigation
```ruby
records_item = NavigationItem.new(
  'Records',
  records_path,
  'file',
  true,
  [
    NavigationItem.new('All', records_path, 'list', true, [], nil),
    NavigationItem.new('New', new_record_path, 'plus', false, [], nil)
  ],
  nil
)
```

### Breadcrumb Trail
```ruby
breadcrumbs = [
  { label: 'Home', path: root_path, icon: 'home', current: false },
  { label: 'Records', path: records_path, icon: 'file', current: false },
  { label: 'My Record', path: record_path(@record), icon: 'document', current: true }
]
```

### Category Navigation with Badges
```ruby
categories = [
  { id: 1, name: 'Mechanical', icon: 'gear', count: 5 },
  { id: 2, name: 'Electrical', icon: 'zap', count: 3 }
]
nav = builder.category_navigation(categories)
```

## Active State Determination

### By Controller/Action
```ruby
builder.is_active?('records')           # Checks controller
builder.is_active?('records', 'index')  # Checks controller and action
```

### By Route Matching
The system analyzes the current request path and parameters:
```ruby
@current_request.path_parameters[:controller]  # 'records'
@current_request.path_parameters[:action]      # 'index'
```

### By Query Parameters
For category filters:
```ruby
builder.is_active_category?(@category_id)
```

## Authorization Integration

### Permission-Based Navigation

authorized_nav = builder.authorized_navigation(current_user)

## Uniform Navigation Principles

This implementation maintains:

1. **Isolation** - Navigation logic isolated in NavigationBuilder service
2. **Clarity** - Clear NavigationItem structure used throughout
3. **Uniformity** - Consistent patterns for all navigation types
4. **Maintainability** - Single source of truth for navigation definitions
5. **Extensibility** - Easy to add new navigation items or patterns
6. **Framework Native** - Uses only Rails built-in APIs

## Extension Points

### Adding New Navigation Items
```ruby
# In NavigationBuilder.main_navigation
NavigationItem.new(
  'New Feature',
  Rails.application.routes.url_helpers.new_feature_path,
  'star',
  is_active?('new_feature'),
  [],
  nil
)
```

### Custom Navigation Rendering
```ruby
# In NavigationHelper
def render_custom_navigation
  builder = NavigationBuilder.new(current_user, request)
  items = builder.main_navigation
  # Custom rendering logic
end
```

### Authorization Integration
```ruby
# Extend is_authorized? in NavigationBuilder
def is_authorized?(user, item_label)
  case item_label
  when 'Admin'
    user&.admin?
  when 'Analytics'
    user&.analytics_access?
  else
    true
  end
end
```
